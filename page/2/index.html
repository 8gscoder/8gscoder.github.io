kubernetes/runtime/scheme

发表于 2020-05-14 更新于 2020-05-24 分类于 kubernetes

scheme

一、前言

k8s 有一篇proposal介绍了Kubernetes API是有apigroup的观念的，即不同group的版本是不断变化的，分开管理，以适应不通的迭代速度（k8s的资源都是有gh SIG进行分开迭代管理的）。这也导致了k8s apiserver的版本管理是神tm复杂，比如k8s的一个版本的release，可能会导致一些资源的version进行变化，比如1.7的rbac是v1beta1，而1.8之后rbac默认就是v1了，你给1.8版本的apiserver提交v1beta1的rbac资源，k8s apiserver会怎么处理呢？因为高版本肯定要像低版本兼容，所以1.8的apiserver肯定也要能兼容v1beta1，毕竟v1beta1也是个beta，你不能突然搞个破坏性的向前不兼容，这样会在集群升级的时候，会有很大的问题，比如之前etcd里面用protobuf存了v1beta1版本的资源，突然apiserver升级了，rbac的版本变成v1，你怎么搞？你总不能读出来不识别吧，搞出问题来吧，所以肯定是要有兼容机制来处理这些可能是必要的更新问题。总而言之，apiserver的版本管理很复杂，如何来实现API的多版本需要面临的兼容、不兼容机制呢？

Mac操作

发表于 2020-05-12 更新于 2020-05-23 分类于 work

1. MAC通用快捷键

mac的编辑文档、选中时、iterm操作时有通用的快捷键，包括在Goland中、sublime、vs code中

  1. ctrl+e、ctrl+a：快速移动到行首行尾
  2. ctrl+f、ctrl+b、ctrl+p、ctrl+n：左移右移上下一行，相当于方向键，但是方向键要右手离开键盘
  3. ctrl+d、ctrl+h：在iterm中光标左删右删，等同于删除键和fn+退格键，goland不适用会和其他键冲突
  4. ctrl+u、ctrl+k：iterm，删除光标之前所以字幕、删除光标之后所有字幕
  5. command+shift+w：企业微信

Terms

  1. Legacy：遗留，即将废弃的意思

  2. Generic：通用的，比如GenericAPIServer，即是通用的APIServer服务，不是仅适用于K8s Apiserver的，比如tke-apiserver就是基于这个做的，底层都是go-restful和http框架

  3. Delegation：委派、代理模型

  4. Paritcular：特定的，指定的，特例化后的
