Mac操作

Posted on 2020-05-12 Edited on 2020-05-23 In work

1. MAC通用快捷键

mac的编辑文档、选中时、iterm操作时有通用的快捷键，包括在Goland中、sublime、vs code中

  1. ctrl+e、ctrl+a：快速移动到行首行尾
  2. ctrl+f、ctrl+b、ctrl+p、ctrl+n：左移右移上下一行，相当于方向键，但是方向键要右手离开键盘
  3. ctrl+d、ctrl+h：在iterm中光标左删右删，等同于删除键和fn+退格键，goland不适用会和其他键冲突
  4. ctrl+u、ctrl+k：iterm，删除光标之前所以字幕、删除光标之后所有字幕
  5. command+shift+w：企业微信

Terms

代码命名中有一些常见的单词，有特别的意思，在这里记录一下

  1. Legacy：遗留，即将废弃的意思

  2. Generic：通用的，比如GenericAPIServer，即是通用的APIServer服务，不是仅适用于K8s Apiserver的，比如tke-apiserver就是基于这个做的，底层都是go-restful和http框架

  4. Paritcular：特定的，指定的，特例化后的

DB troubleshooting

Posted on 2020-05-10 Edited on 2020-05-23 In mysql

DB 高负载

  • 连接数超限
  • 流量大
  • cpu高负载
  • 慢查询

常用命令：

慢查询

一、特征

  1. CPU负载高

  2. Server端读写DB的IO负载高，Server端阻塞

  3. SQL延迟高

二、相关配置

  1. 查询mysql配置： show variables like %query%;
  2. 临时开启慢查询配置：
    1. set global slow_query_log = on;打开开关
    2. set long_query_time = 1; 慢查询阈值
    3. set globle log_output = file;慢查询存储方式
    4. show variables like 'max_connections';
    5. 永久开起，需在配置文件中开启

二、排查思路

  1. 现网的DB都是默认开启来慢查询log的（mysql.cnf：slow_query_log，long_query_time)

  2. 查看慢查询日志

    1. mysqldumpslow

      1. mysqldumpslow -s c -t 10 -a -l

        1. -s指定排序模式

          1. al: average lock time
ar: average rows sent
at: average query time
c: count
l: lock time
r: rows sent
t: query time

        2. -t

        3. -a 显示具体sql

        4. -g 过滤

        5. -l 总时间中包括锁的时间

三、如何优化

排查如果是因为未建索引导致，根据业务需要加上索引

四、case记录

  1. 内部用户并发删除资源，上层逻辑没有限频，导致DB慢查询，而由于一个旁路业务连接数异常较高，导致连接数较高，慢查询导致并发连接数升高，导致达到连接数最大值，导致无法建立新连接，从而导致雪崩。
