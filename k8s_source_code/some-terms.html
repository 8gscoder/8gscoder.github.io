代码命名中有一些常见的单词，有特别的意思，在这里记录一下

  1. Legacy：遗留，即将废弃的意思

  2. Generic：通用的，比如GenericAPIServer，即是通用的APIServer服务，不是仅适用于K8s Apiserver的，比如tke-apiserver就是基于这个做的，底层都是go-restful和http框架

  3. Delegation：委派、代理模型

  4. Paritcular：特定的，指定的，特例化后的

k8s常用的数据结构

  1. XX-Storage： 存储对象，比如apiserver的各种Storage封装了对后端etcd的操作
  2. Option、Config、Server：Option分装命令行参数，Config分装配置参数，Server分装服务端对象
  3. Aggregator：聚合
  4. Extensions：扩展的

k8s常用的设计模式：

  1. 工厂模式
    1. 以New为开头的接收器方法，基本上都是运用了工厂模式
    2. New一个Factory，Factory有CreateXX(opt *Options)方法，依据传入不同的Option，生产出不通的类型，比如RESTGetter、StorageGetter，传入不通的resource schema，产生不同的server/storage..
    3. 其他参数options可能会有ApplyTo(c *generic.Config)方法，会把自己赋值到generic.Config上
    4. 最后生产出来一个particular的产物
  2. 装饰器模式
    1. WithAudit、WithXXX，都是用到了装饰器模式
